# 마을봇

마을봇은 팀 운영에 필요한 반복 알림을 자동으로 전송해 주는 Mattermost 알림 봇입니다.  
GitHub Actions의 스케줄 기능을 기반으로 동작하며, 정해진 시간에 출석 체크, 스크럼, 스트레칭, 회고 등 팀 루틴 메시지를 자동 발송합니다.

단순히 알림을 보내는 데서 그치지 않고, 팀원들이 놓치기 쉬운 일정을 일정한 톤으로 안내해 팀 운영의 흐름을 안정적으로 유지하는 것을 목표로 합니다.

---

## 주요 기능

- **정해진 시간 자동 알림 발송**
  - 입실 체크
  - 아침 스크럼
  - 스트레칭 안내
  - 회고 알림

- **GitHub Actions 기반 스케줄 실행**
  - 별도 서버 없이 GitHub Actions cron으로 주기 실행
  - 평일 기준 반복 운영 가능

- **Mattermost 웹훅 연동**
  - 팀 메신저 채널에 자동 메시지 전송
  - 운영 공지를 한 채널에서 일관되게 관리 가능

- **수동 테스트 지원**
  - `workflow_dispatch`를 통해 특정 알림만 개별 테스트 가능
  - 실제 운영 전 메시지 형식과 동작 검증 가능

- **운영 안정성 고려**
  - schedule 이벤트일 때만 실제 발송되도록 분기 처리
  - 테스트 실행 시 불필요한 실제 발송 방지 가능

---

## 동작 방식

1. GitHub Actions가 설정된 cron 시간에 실행됩니다.
2. 현재 실행된 스케줄에 맞는 알림 종류를 판별합니다.
3. 해당 메시지를 Mattermost Incoming Webhook으로 전송합니다.
4. 팀원들은 정해진 시간에 맞춰 운영 알림을 받을 수 있습니다.

---

## 사용 기술

- **GitHub Actions**
  - 정해진 시간에 워크플로우 실행
- **Mattermost Incoming Webhook**
  - 메시지 전송
- **Shell Script / curl**
  - 웹훅 호출 및 메시지 발송
- **Cron**
  - 반복 스케줄 관리

---

## 예시 알림

- **입실 체크**  
  하루 시작 시 출석 및 업무 시작 유도

- **스크럼 알림**  
  오전 일정 공유 및 팀 싱크 맞춤

- **스트레칭 알림(현재 프로젝트 집중 기간으로 비활성화)**  
  장시간 앉아 있는 팀원들을 위한 중간 환기

- **회고 알림**  
  하루 작업 정리 및 기록 유도

---

## 기대 효과

- 반복적인 운영 공지의 수동 부담 감소
- 일정 누락 방지 및 팀 루틴 정착
- 팀 커뮤니케이션의 일관성 확보
- 운영 업무 자동화를 통한 협업 효율 향상

---

## 실행 환경

해당 봇은 별도의 상시 서버 없이 GitHub 저장소 내에서 운영됩니다.  

---

## 향후 개선 방향

- 알림 종류별 메시지 템플릿 분리
- 랜덤 스트레칭 문구/루틴 제공
- 공휴일 및 예외 일정 분기 처리
- 팀 상황에 따라 알림 시간 유연 조정
- 발송 이력 관리 및 실패 로그 확인 기능 보완

---

## 프로젝트 의의

마을봇은 단순한 알림 자동화 도구가 아니라,  
팀이 매일 반복하는 운영 흐름을 안정적으로 유지하도록 돕는 협업 보조 봇입니다.

반복되는 공지를 자동화함으로써 팀원들은 더 중요한 업무에 집중할 수 있고,  
운영자는 매번 직접 안내하지 않아도 팀의 기본 리듬을 유지할 수 있습니다.
