# 마을봇(maeulbot)

> GitHub Actions와 Mattermost 웹훅을 활용해 팀 운영 알림을 자동화한 협업 보조 봇

마을봇은 팀의 반복적인 운영 공지를 자동화하기 위해 만든 알림 봇입니다.  
출석 체크, 스크럼, 스트레칭, 회고 등 매일 반복되는 팀 루틴 메시지를 정해진 시간에 자동 발송하도록 구성했습니다.

수동으로 공지를 전달하던 방식은 일정 누락이나 운영 부담이 발생하기 쉬웠기 때문에,  
GitHub Actions의 스케줄 기능과 Mattermost Incoming Webhook을 연동해 별도 서버 없이 안정적으로 동작하는 자동화 흐름을 구현했습니다.

## 프로젝트 개요

- **프로젝트명**: 마을봇
- **유형**: 팀 운영 자동화 봇
- **목적**: 반복적인 운영 공지 자동화를 통한 협업 효율 향상
- **주요 사용자**: 프로젝트 팀원(E106 마을회관)
- **운영 기간**: 2026.02 ~ 2026.04

## 담당 및 구현 내용

- GitHub Actions 기반 정기 실행 워크플로우 구성
- Mattermost Incoming Webhook 연동을 통한 자동 메시지 발송 구현
- 입실, 스크럼, 스트레칭, 회고 등 팀 루틴별 알림 분기 처리
- `workflow_dispatch` 기반 수동 테스트 환경 구성
- 테스트 실행과 실제 스케줄 실행을 분리해 불필요한 발송 방지
- 팀 운영 흐름에 맞춰 알림 시간 및 메시지 형식 조정

## 주요 기능

- **정기 알림 자동 발송**
  - 출석 체크
  - 스크럼 알림
  - 스트레칭 안내
  - 회고 알림

- **서버 없는 자동화 운영**
  - GitHub Actions cron 기반 실행
  - 별도 인프라 없이 저장소 내 운영 가능

- **협업 도구 연동**
  - Mattermost 채널에 자동 공지 전송
  - 팀 커뮤니케이션 채널 내 일관된 운영 가능

- **운영 안정성 고려**
  - schedule 이벤트에서만 실제 발송되도록 분기 처리
  - 수동 테스트 시 운영 메시지 오발송 방지

## 사용 기술

- **GitHub Actions**
- **Mattermost Incoming Webhook**
- **Shell Script**
- **curl**
- **cron**

## 동작 방식

1. GitHub Actions가 설정된 시간에 워크플로우를 실행합니다.
2. 현재 실행 시점에 맞는 알림 유형을 판별합니다.
3. 해당 메시지를 Mattermost Webhook으로 전송합니다.
4. 팀원들은 정해진 시간에 맞춰 운영 알림을 받게 됩니다.

## 성과 및 의의

- 반복적인 운영 공지의 수동 부담을 줄여 팀 운영 효율 향상
- 출석, 스크럼, 회고 등 핵심 루틴의 누락 가능성 감소
- 팀 커뮤니케이션을 일정한 형식으로 표준화
- 별도 서버 없이도 운영 가능한 경량 자동화 구조 구현

## 프로젝트를 통해 드러나는 역량

- 반복 업무를 자동화 흐름으로 전환하는 문제 해결 능력
- 팀 운영 방식에 맞춘 실용적 기능 설계 역량
- 협업 도구와 스케줄링 시스템을 연결하는 구현 능력
- 테스트/운영 환경을 구분해 안정성을 고려한 개발 역량
